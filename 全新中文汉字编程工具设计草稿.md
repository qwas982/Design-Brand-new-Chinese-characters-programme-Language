# 一种基于汉语主谓宾结构与Forth哲学的图灵完备编程语言设计

## 摘要
本文提出一种融合自然语言主谓宾（SVO）语序与Forth栈哲学的新型编程语言。该语言以汉语为蓝本，每个句子严格遵循“主语-谓语-宾语”结构，通过多个栈（操作数栈、操作符栈、上下文栈、环境栈）进行解析，动态构建主宾谓（主语-宾语-谓语）形式的递归树，最终后序遍历生成琼斯Forth代码，可运行于寄存器机器。本文详细阐述了文法设计、解析算法、树结构及代码生成过程，并用示例展示了数据类型、控制流、函数定义等图灵完备要素的实现。

## 1. 引言
传统编程语言的语法往往与自然语言相去甚远，学习曲线陡峭。Forth语言以其极简的栈机制和后缀表示法（操作数在前，操作符在后）著称，但其语法与自然语言仍有差距。汉语作为典型的SVO语言，主语、谓语、宾语的顺序与Forth的主宾谓（SOV）顺序恰好形成镜像关系。本文尝试将两者结合，设计一种既符合汉语习惯又继承Forth哲学的语言，使代码读起来像中文句子，同时保持编译的简洁性。

## 2. 核心思想
### 2.1 主谓宾到主宾谓的映射
自然语言句子结构：主语 谓语 宾语  
Forth哲学：操作数先行，操作符后行，即 主语 宾语 谓语  
因此，我们只需将谓语后置即可得到Forth风格的计算顺序。例如：
- 汉语：`猫 吃 鱼`  
- 主宾谓：`猫 鱼 吃`  
在解析过程中，我们通过栈动态实现这种转换，并构建递归树。

### 2.2 嵌套与递归树
为表达复杂逻辑，允许主语或宾语为子句（用括号括起的句子），形成嵌套树。每个节点统一表示为 `(主语 宾语 谓语)`，其中主语和宾语可以是原子或子节点，谓语为动词/助词。这种表示法提取了S表达式的树形灵魂，但采用主宾谓顺序，完全符合Forth哲学。

### 2.3 汉语助词的融入
汉语中的助词（如“的”、“若”、“当”、“了”、“函”、“则”、“否则”）被纳入谓语体系，赋予确定语义，使语言表达更自然。例如：
- “的”表示属性访问或函数调用（参数 的 函数名）
- “若”构成条件句（主语 若 谓语 则 真句组 否则 假句组）
- “当”构成循环句（主语 当 谓语 则 循环体）
- “了”表示动作完成，隐式赋值（主语 谓语 宾语 了）

## 3. 文法设计（EBNF）
采用上下文无关文法，严格遵循主语-谓语-宾语语序，以句号结尾。列表用逗号分隔，子句用括号括起。

```
程序        = { 句子 } 。

句子        = 普通句
            | 条件句
            | 循环句
            | 函数定义
            | 返回句 。

普通句      = 主语 谓语 [ 宾语 ] [ "了" ] "。" 。

条件句      = 主语 "若" 谓语 [ 宾语 ] "则" 句组 [ "否则" 句组 ] "。" 。

循环句      = 主语 "当" 谓语 [ 宾语 ] "则" 句组 "。" 。

函数定义    = 标识符 "函" "参数" 参数列表 "则" 句组 "。" 。

返回句      = 表达式 "返回" "。" 。

主语        = 原子
            | 列表
            | "(" 句子 ")" 。

宾语        = 原子
            | 列表
            | "(" 句子 ")"
            | 谓语短语 。

谓语短语    = 谓语 [ 宾语 ] 。        /* 用于条件/循环中缺少主语的谓语部分 */

原子        = 标识符
            | 数字
            | 字符串 。

列表        = 元素 { "，" 元素 } 。   /* 逗号分隔，作为一个整体 */

元素        = 原子
            | "(" 句子 ")" 。

参数列表    = 标识符 { "，" 标识符 } 。

句组        = 句子 { "，" 句子 } 。   /* 多个句子，逗号分隔，最后一个无句号 */

谓语        = 动词
            | 助词
            | 函数名
            | 类型名
            | 字面量 。

动词        = "加" | "减" | "乘" | "除" | "取余"
            | "等于" | "大于" | "小于" | "不等于" | "大于等于" | "小于等于"
            | "与" | "或" | "非"
            | "设为" | "取" | "存" | "输出" | "输入"
            | "为" | "含" | "有"
            | "地址" | "值"            /* 指针操作 */
            | 用户自定义动词 。

助词        = "的" | "若" | "当" | "函" | "返回" | "是" | "则" | "否则" | "参数" 。

标识符      = 汉字 | 字母 ( 汉字 | 字母 | 数字 )* 。   /* 允许中文标识符 */

数字        = 整数 | 浮点数 。
字符串      = " " " { 任意字符 } " " " 。
```

## 4. 多栈解析算法
解析器使用四个栈协同工作，线性扫描token序列，逐步归约为主宾谓树。

### 4.1 四个栈的功能
| 栈名       | 作用                                                                 |
|------------|----------------------------------------------------------------------|
| 操作数栈   | 存放已解析的原子、子句节点、列表节点等，供归约时弹出。               |
| 操作符栈   | 存放待归约的谓语，每个条目记录谓语名和所需参数个数。                 |
| 上下文栈   | 保存和恢复解析状态，处理括号、句组等嵌套结构。                       |
| 环境栈     | 维护符号表作用域链，记录变量、函数定义，用于语义检查和代码生成。     |

### 4.2 解析规则
- **原子**：压入操作数栈。
- **谓语**：压入操作符栈；若栈顶谓语所需参数齐备，则立即归约。
- **逗号**：作为二元谓语，构建列表节点 `(左 右 逗号)`。
- **“则”、“否则”**：触发句组开始/结束，通过上下文栈保存当前状态，收集句组句子形成列表节点。
- **括号“（”**：压入上下文栈，开始新子句；遇到“）”时弹出上下文，将子句结果压回上层操作数栈。
- **句号**：触发当前句子归约，弹出操作符栈中所有谓语，最终操作数栈顶即为句子根节点，将其移出作为一棵树，清空操作数栈准备下一句子。

### 4.3 示例解析：条件句
输入：`x 若 大于 y 则 x 输出 否则 y 输出 。`
解析过程关键步骤见表1。

**表1 条件句解析过程**
| 步骤 | 当前词 | 操作数栈              | 操作符栈          | 动作说明                         |
|------|--------|-----------------------|-------------------|----------------------------------|
| 1    | x      | [x]                   | []                | 压入x                           |
| 2    | 若     | [x]                   | [若(需3)]         | 压入若                           |
| 3    | 大于   | [x]                   | [若(需3), 大于(需2)] | 压入大于                         |
| 4    | y      | [x, y]                | [若(需3), 大于(需2)] | 压入y                           |
| 5    | 归约   | [(x y 大于)]          | [若(需3)]         | 大于齐备，归约为(x y 大于)       |
| 6    | 则     | 保存状态，开始真句组  |                   | 压入上下文                       |
| 7    | x      | [x]                   | []                | 新操作数栈                       |
| 8    | 输出   | [(x () 输出)]         | []                | 归约为(x () 输出)                |
| 9    | 否则   | 真句组结束，恢复状态  |                   | 真句组列表=[(x () 输出)]，开始假句组 |
| 10   | y      | [y]                   | []                |                                 |
| 11   | 输出   | [(y () 输出)]         | []                | 归约为(y () 输出)                |
| 12   | 。     | 假句组结束，恢复状态  |                   | 假句组列表=[(y () 输出)]         |
| 13   | 归约若 | 弹出若及三操作数      |                   | 得((x y 大于) (真 假) 若)        |

## 5. 递归树结构
每个节点采用 `(主语 宾语 谓语)` 形式，主语和宾语可为：
- 原子（标识符、数字）
- 子节点（另一棵树）
- 列表节点（由逗号构建的二叉树，表示参数序列）
- 句组节点（由“则”构建的列表，表示多个顺序句子）

程序由多个顶层树组成，按顺序执行。

### 5.1 示例树
阶乘函数定义：
```
阶乘 函 参数 n 则
    n 若 等于 0 则 1 返回 否则 n 乘 (n 减 1) 的 阶乘 返回
。
```
解析得到的主宾谓树（简化）：
```
(阶乘 
   ( (n) 
     ( (n 0 等于) 
         ( (1 返回) 
           ( (n (n 1 减) 乘) 阶乘 的 返回 ) 
         若 ) ) 
   函)
```

## 6. 代码生成：后序遍历生成琼斯Forth
后序遍历每棵树，根据谓语映射生成Forth代码，并利用环境栈处理变量作用域和函数调用。

### 6.1 映射规则
| 谓语 | 生成代码（后序） | 说明                         |
|------|------------------|------------------------------|
| 数字 | 直接压栈         | 例如 `5`                     |
| 标识符| 若为右值：`@`    | 全局变量需先取地址再`@`      |
| 加   | `+`              | 弹出两数相加                 |
| 大于 | `>`              | 对比                         |
| 设为 | `!`              | 值压栈，地址压栈，然后`!`    |
| 的   | 如 `a i @ + @`   | 数组元素访问                 |
| 若   | `IF ... ELSE ... THEN` | 条件分支                 |
| 当   | `BEGIN ... WHILE ... REPEAT` | 循环                 |
| 函   | `: 函数名 ... ;`  | 函数定义                     |
| 返回 | 不生成代码，值已在栈顶 | 由`;`结束函数             |
| 输出 | `.`               | 打印栈顶                     |

### 6.2 示例：阶乘函数代码生成
后序遍历阶乘树，得到以下Forth代码：
```forth
: factorial ( n -- n! )
   dup 0= IF drop 1 ELSE dup 1 - recurse * THEN ;
```
其中`recurse`是Forth中递归调用自身的关键词。

### 6.3 完整程序示例
输入程序：
```
阶乘 函 参数 n 则
    n 若 等于 0 则 1 返回 否则 n 乘 (n 减 1) 的 阶乘 返回
。
结果 设为 5 阶乘 。
结果 输出 。
```
生成Forth代码：
```forth
: factorial ( n -- n! )
   dup 0= IF drop 1 ELSE dup 1 - factorial * THEN ;
5 factorial
结果 !
结果 @ .
```

## 7. 图灵完备性
该语言支持：
- 算术运算（加、减、乘、除等）
- 对比与逻辑运算（大于、等于、与、或、非）
- 条件分支（若）
- 循环（当）
- 函数定义与递归（函）
- 数据结构（数组、结构体、指针）
- 输入输出（输入、输出）

因此可以模拟任意图灵机，是图灵完备的。

## 8. 结论
本文设计了一种基于汉语主谓宾语序与Forth栈哲学的新语言。通过四个栈的协同解析，将自然语言句子动态归约为主宾谓递归树，再后序遍历生成琼斯Forth代码。该方法既保留了汉语的自然表达，又继承了Forth的简洁与高效，为领域特定语言和教学语言的设计提供了新思路。未来工作可扩展标准库、优化代码生成，并实现自举编译器。
