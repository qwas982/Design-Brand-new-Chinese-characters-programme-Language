# 中C语言参考手册：句法与词法标准

## 第一章 概述

中C是一门**编译型、静态类型、系统级编程语言**。它在保留C语言全部底层能力与灵活性的基础上，对词法、句法进行了全盘中文化重构，并引入了现代化的模块系统与异常处理机制。

**核心设计原则**：
1.  **完全中文化**：所有关键字、类型名、操作符均为中文，消除英文语言障碍。
2.  **与C等价的能力**：不剥夺程序员对内存、指针的精细控制权，保留C语言“所见即所得”的性能模型。
3.  **无预处理器**：采用Python式的逻辑模块导入系统，避免宏文本替换的副作用。
4.  **结构化异常处理**：内置 `尝试`/`捕获`/`抛出`，提升错误处理代码的可读性与健壮性。

**未定义行为的处理**：本标准未详尽描述的语义（如求值顺序、类型转换规则、未定义行为列表等），默认遵循 **ISO/IEC 9899:2017 (C17)** 标准。

## 第二章 词法结构

### 2.1 字符集与源文件编码

中C源文件**必须**使用 **UTF-16BE** 编码。编译器前端需把输入流解码为Unicode码点序列。

### 2.2 空白与注释

空白字符（空格、制表符 `\t`、换行符 `\n`、回车符 `\r`）用于分隔词法单元，本身无语义。

注释沿用C风格：
- **单行注释**：`//` 直至行尾的所有字符视为注释。
- **多行注释**：`/*` 与 `*/` 之间的所有字符视为注释，不支持嵌套。

### 2.3 标识符

标识符用于命名变量、函数、结构体、枚举等用户自定义符号。

**规则**：
- 首字符：**中文字符**、**英文字母**（`a-z`、`A-Z`）、**下划线**（`_`）。
- 后续字符：上述字符及**数字**（`0-9`）。
- 不允许与**保留关键字**（见2.5节）重名。

> **标准说明**：中文字符范围定义为Unicode汉字区块（CJK Unified Ideographs, U+4E00 至 U+9FFF）及扩展区块。编译器实现时建议采用ICU库或类似方案处理Unicode字符属性。

### 2.4 字面常量

| 类型 | 词法规约 | 示例 |
| :--- | :--- | :--- |
| **整数字面量** | 十进制：`[1-9][0-9]*`<br>二进制：`0[bB][0-1]*`<br>十六进制：`0[xX][0-9a-fA-F]+` | `123`、`0b0011`、`0xFF` |
| **浮点字面量** | 包含小数点或指数符号（`e`/`E`），后缀可选 `f`/`F`（浮32） | `3.14`、`2.5e-3f` |
| **字符字面量** | 单引号包围单个字符或转义序列 | `'A'`、`'中'`、`'\n'` |
| **字符串字面量** | 双引号包围字符序列，自动连接相邻字符串 | `"你好"`、`"世界"` |
| **布尔字面量** | 内置常量 | `真`、`假` |
| **空指针字面量** | 内置常量 | `空指针` |

### 2.5 保留关键字（完整列表）

以下单词保留为语言关键字，**不可**用作标识符。

#### 类型关键字
`空`、`逻辑`、`整8`、`整16`、`整32`、`整64`、`整128`、`无整8`、`无整16`、`无整32`、`无整64`、`无整128`、`浮16`、`浮32`、`浮64`、`浮128`、`字8`、`字16`、`字32`、`字128`、`指针`、`仅读指针`、`结构`、`枚举`、`联合`

#### 控制流关键字
`若`、`否则`、`否则若`、`一会儿`、`为`、`遍历`、`继续`、`破断`、`返回`

#### 异常处理关键字
`尝试`、`捕获`、`抛出`、`最终`

#### 内存管理关键字
`分配`、`释放`

#### 模块系统关键字
`导入`、`从`、`为`

#### 操作符别名关键字
`求大小`、`取址`、`读针`、`写针`、`与`、`或`

#### 字面常量关键字
`真`、`假`、`空指针`

### 2.6 分隔符与运算符

中C保留了C语言的所有标点符号，仅增加中文关键字作为操作符替代。

| 类别 | 符号 |
| :--- | :--- |
| **算术** | `+` `-` `*` `/` `%` |
| **关系** | `<` `>` `<=` `>=` `==` `!=` |
| **逻辑** | `与`（等价 `&&`）、`或`（等价 `\|\|`）、`非`（等价 `!`） |
| **位运算** | `&` `\|` `^` `~` `<<` `>>` |
| **赋值** | `=` `+=` `-=` `*=` `/=` `%=` `&=` `\|=` `^=` `<<=` `>>=` |
| **成员访问** | `.` `->` |
| **其他** | `(` `)` `[` `]` `{` `}` `,` `;` `:` `?` |

## 第三章 句法标准（扩展BNF）

本章给出中C句法的完整产生式。采用 **EBNF** 表示法：
- `{ X }` 表示 X 出现零次或多次。
- `[ X ]` 表示 X 可选。
- `( X | Y )` 表示选择。
- 中文关键字直接书写。

### 3.1 程序结构

```ebnf
<翻译单元>           ::= { <模块导入声明> } { <顶层声明> }

<模块导入声明>       ::= "导入" <模块路径> [ "为" <标识符> ] ";"
                       | "从" <模块路径> "导入" <导入项列表> ";"

<模块路径>           ::= <字符串字面量>

<导入项列表>         ::= <标识符>
                       | "{" <标识符> { "," <标识符> } "}"

<顶层声明>           ::= <函数定义>
                       | <全局变量声明>
                       | <类型定义>
                       | ";"                           (* 空声明 *)
```

### 3.2 类型系统

```ebnf
<类型>               ::= <基本类型>
                       | <指针类型>
                       | <仅读指针类型>
                       | <数组类型>
                       | <结构类型>
                       | <枚举类型>
                       | <联合类型>
                       | <函数指针类型>
                       | "(" <类型> ")"                (* 括号类型 *)

<基本类型>           ::= "空"
                       | "逻辑"
                       | <整数类型>
                       | <浮点类型>
                       | <字符类型>

<整数类型>           ::= "整8" | "整16" | "整32" | "整64" | "整128"
                       | "无整8" | "无整16" | "无整32" | "无整64" | "无整128"

<浮点类型>           ::= "浮16" | "浮32" | "浮64" | "浮128"

<字符类型>           ::= "字8" | "字16" | "字32" | "字128"

<指针类型>           ::= "指针" "<" <类型> ">"
<仅读指针类型>       ::= "仅读指针" "<" <类型> ">"

<数组类型>           ::= <类型> "[" [ <常量表达式> ] "]"

<结构类型>           ::= "结构" <标识符> [ "{" { <类型> <标识符> ";" } "}" ]

<枚举类型>           ::= "枚举" <标识符> [ "{" <枚举项> { "," <枚举项> } "}" ]

<联合类型>           ::= "联合" <标识符> [ "{" { <类型> <标识符> ";" } "}" ]

<枚举项>             ::= <标识符> [ "=" <常量表达式> ]

<函数指针类型>       ::= <类型> "(" "*" <标识符> ")" "(" [ <类型列表> ] ")"

<类型列表>           ::= <类型> { "," <类型> }
```

### 3.3 声明与定义

```ebnf
<全局变量声明>       ::= <类型> <标识符> [ "=" <初始值> ] ";"
<局部变量声明>       ::= <类型> <标识符> [ "=" <初始值> ] ";"

<初始值>             ::= <赋值表达式>
                       | "{" <初始值列表> "}"
<初始值列表>         ::= <初始值> { "," <初始值> }

<函数定义>           ::= <类型> <标识符> "(" [ <参数列表> ] ")" <代码块>

<参数列表>           ::= <参数> { "," <参数> }
<参数>               ::= <类型> <标识符>

<类型定义>           ::= "类型" <标识符> "=" <类型> ";"   (* 相当于C的typedef *)
```

### 3.4 语句

```ebnf
<语句>               ::= <表达式语句>
                       | <代码块>
                       | <条件语句>
                       | <循环语句>
                       | <跳转语句>
                       | <异常处理语句>
                       | <内存管理语句>
                       | <局部变量声明>
                       | ";"                           (* 空语句 *)

<表达式语句>         ::= <表达式> ";"

<代码块>             ::= "{" { <语句> } "}"

<条件语句>           ::= "若" "(" <表达式> ")" <语句> [ "否则" <语句> ]
                       | "若" "(" <表达式> ")" <语句> { "否则若" "(" <表达式> ")" <语句> } [ "否则" <语句> ]

<循环语句>           ::= "一会儿" "(" <表达式> ")" <语句>
                       | "为" "(" [ <表达式> ] ";" [ <表达式> ] ";" [ <表达式> ] ")" <语句>

<跳转语句>           ::= "继续" ";"
                       | "破断" ";"
                       | "返回" [ <表达式> ] ";"

<异常处理语句>       ::= "尝试" <代码块> <捕获子句> [ "最终" <代码块> ]

<捕获子句>           ::= "捕获" "(" <类型> <标识符> ")" <代码块>
                       | "捕获" "(" "..." ")" <代码块>

<内存管理语句>       ::= <表达式> ";"                  (* 分配/释放作为表达式使用 *)
```

### 3.5 表达式

中C表达式句法与C高度一致，优先级和结合性相同。

```ebnf
<表达式>             ::= <赋值表达式>

<赋值表达式>         ::= <条件表达式>
                       | <单目表达式> <赋值运算符> <赋值表达式>

<赋值运算符>         ::= "=" | "+=" | "-=" | "*=" | "/=" | "%=" | "<<=" | ">>=" | "&=" | "^=" | "|="

<条件表达式>         ::= <逻辑或表达式> [ "?" <表达式> ":" <条件表达式> ]

<逻辑或表达式>       ::= <逻辑与表达式> { "或" <逻辑与表达式> }

<逻辑与表达式>       ::= <按位或表达式> { "与" <按位或表达式> }

<按位或表达式>       ::= <按位异或表达式> { "|" <按位异或表达式> }

<按位异或表达式>     ::= <按位与表达式> { "^" <按位与表达式> }

<按位与表达式>       ::= <相等表达式> { "&" <相等表达式> }

<相等表达式>         ::= <关系表达式> { ( "==" | "!=" ) <关系表达式> }

<关系表达式>         ::= <移位表达式> { ( "<" | ">" | "<=" | ">=" ) <移位表达式> }

<移位表达式>         ::= <加减表达式> { ( "<<" | ">>" ) <加减表达式> }

<加减表达式>         ::= <乘除表达式> { ( "+" | "-" ) <乘除表达式> }

<乘除表达式>         ::= <强制转换表达式> { ( "*" | "/" | "%" ) <强制转换表达式> }

<强制转换表达式>     ::= <单目表达式>
                       | "(" <类型> ")" <强制转换表达式>

<单目表达式>         ::= <后缀表达式>
                       | <单目运算符> <单目表达式>
                       | "求大小" "(" <类型> ")"
                       | "求大小" <单目表达式>
                       | "取址" "(" <单目表达式> ")"
                       | "读针" "(" <单目表达式> ")"
                       | "写针" "(" <单目表达式> "," <表达式> ")"

<单目运算符>         ::= "+" | "-" | "~" | "!"

<后缀表达式>         ::= <基础表达式> { <后缀操作> }

<后缀操作>           ::= "[" <表达式> "]"
                       | "(" [ <实参列表> ] ")"
                       | "." <标识符>
                       | "->" <标识符>
                       | "++"
                       | "--"

<基础表达式>         ::= <标识符>
                       | <常量>
                       | <字符串字面量>
                       | "(" <表达式> ")"
                       | "抛出" <表达式>
                       | "分配" "(" <表达式> ")"          (* 分配原始字节 *)
                       | "释放" "(" <表达式> ")"          (* 释放内存 *)

<实参列表>           ::= <表达式> { "," <表达式> }

<常量>               ::= <整数字面量>
                       | <浮点字面量>
                       | <字符字面量>
                       | "真"
                       | "假"
                       | "空指针"
```

## 第四章 语义补充（依据C17标准）

以下句法结构未在上文BNF中明确定义，但属于C语言范畴。在中C编译器中，应按**ISO C17标准**实现。

### 4.1 作用域与链接
- 全局标识符默认具有**外部链接**（`extern`），使用 `静态` 关键字限定内部链接。
- 局部变量默认为**自动存储期**，可使用 `静态` 改变存储期。
- 模块导入语义：`导入` 类似于 C `#include` 但具有命名空间隔离；具体链接机制由实现定义，但不得破坏C标准要求的独立编译模型。

### 4.2 类型兼容性与转换
- 算术类型之间的隐式转换规则（整型提升、算术转换）与C完全相同。
- 指针与整数的转换、`空` 指针的赋值规则遵循C标准。
- `仅读指针<T>` 与 `指针<T>` 之间的赋值兼容性：允许把 `指针<T>` 赋值给 `仅读指针<T>`，反之需要显式强制转换（且可能导致未定义行为）。

### 4.3 未定义行为列表
C17 附录 J.2 所列的未定义行为，凡适用于中C相应语言结构的，均应视为未定义行为。编译器前端无需诊断，后端应遵循“不产生额外开销”的原则处理。

### 4.4 属性与扩展
中C保留 `[[]]` 属性句法，用于未来扩展（如 `[[抛弃]]` 标记函数返回值不可忽略）。当前版本不定义任何标准属性。

## 第五章 编译器前端实现指引

### 5.1 词法分析建议
- 由于关键字全为中文字符，建议使用 **双缓冲或多字节解码器** 读取UTF-16BE流。
- 汉字关键字识别可采用 **Trie树** 以提升匹配速度。

### 5.2 句法分析建议
- 本句法无二义性，适合采用 **递归下降解析器（Recursive Descent Parser）**。
- 由于中C表达式句法与C等价，可直接复用成熟C编译器前端的 **运算符优先级解析算法**。

### 5.3 抽象句法树设计
建议为每个产生式定义对应的AST节点。特别注意：
- `取址`、`读针`、`写针` 应映射为 `UnaryOperator` 节点。
- `分配`、`释放` 映射为 `CallExpr` 节点，但其内部链接特殊的内置函数。
- `尝试`/`捕获` 需生成控制流图特殊边，以支持栈展开。
